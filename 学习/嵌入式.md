# SmartFan 设计说明（基于 embARC Baremetal 示例）

  

## 一、设计背景

  

随着小型嵌入式设备与物联网应用的普及，风扇作为常见执行器被广泛用于散热与环境调节。传统“固定转速/手动开关”的方式难以在噪声、能耗与散热之间取得平衡；同时，用户也希望在不接触设备的情况下进行远程控制。

  

本项目以 embARC 平台（IoTDK）为硬件基础，综合使用硬件 PWM、DS18B20 温度传感器与 RN4020 BLE 模块，完成一个“本地按键 + 蓝牙远程 + 温控自动调速”的智能风扇控制示例，强调在裸机环境下的可移植性、实时性与工程化结构。

  

## 二、设计要求

  

### 2.1 功能性要求

  

1. 风扇转速控制

   - 采用硬件 PWM 输出调速。

   - 手动档位提供 3 档：30%、80%、100%。

   - 上电默认关闭风扇（占空比 0%），避免下载/复位后立即转动。

  

2. 本地按键交互

   - 按键1（GPIO IO2）：

     - 短按：手动模式下循环切换档位（30% → 80% → 100% → 30%）。

     - 长按：风扇开/关（toggle）。

   - 按键2（GPIO IO0）：

     - 短按：切换手动/自动模式。

  

3. 自动温控模式

   - 使用 DS18B20 读取温度（0.1℃）。

   - 采样周期约 1s。

   - 温度到占空比映射：

     - $T \le 10\,^{\circ}\mathrm{C}$：0%

     - $T \ge 30\,^{\circ}\mathrm{C}$：100%

     - 10～30℃：线性映射到 0～100%

  

4. 蓝牙远程控制

   - 通过 RN4020 BLE 模块接收小程序写入命令。

   - 命令映射：

     - `1`：风扇开/关（toggle）

     - `2`：切换档位（循环 30/80/100，仅手动模式下生效）

     - `3`：切换手动/自动模式

   - 蓝牙输入与按键输入统一汇总到同一控制状态机。

  

### 2.2 性能与工程性要求

  

- 按键消抖：最短有效按下时间约 20ms。

- 长按判定：约 1000ms。

- 主循环使用 1ms 轮询节拍进行计时与边沿识别。

- 蓝牙命令优先级高于按键（保证远程控制响应）。

- 代码模块职责清晰：初始化、PWM 输出、按键事件、1-Wire 时序、BLE 协议解析解耦。

  

## 三、总体方案设计

  

### 3.1 硬件平台与资源分配

  

- 平台：embARC / BOARD=iotdk，BD_VER=10，CUR_CORE=arcem9d（GNU 工具链）。

- 外设资源：

  

| 功能 | 资源/外设 | 板级引脚/说明 |

|---|---|---|

| 风扇 PWM 输出 | DW_PWM_TIMER_0，通道 0 | Arduino D3（PWM0） |

| 按键1输入 | DFSS_GPIO_4B2 bit2 | Arduino IO2（高电平表示按下） |

| 按键2输入 | DFSS_GPIO_4B2 bit0 | Arduino IO0（高电平表示按下） |

| DS18B20 数据线 | DFSS_GPIO_4B2 bit1 | Arduino IO1（GPIO 模式，1-Wire） |

| RN4020 通信 | DFSS_UART_3 | UART3（RN4020 驱动使用 ez_sio 非阻塞读） |

| RN4020 控制脚 | DFSS_GPIO_4B0 若干 | WAKE_SW/WAKE_HW/CMD |

  

说明：DS18B20 1-Wire 总线需要上拉电阻（常见 4.7kΩ），本设计通过“输出拉低 + 输入释放”模拟开漏时序。

  

### 3.2 软件架构与数据流

  

整体采用“初始化 + 单循环状态机”的裸机结构：

  

1. 初始化阶段

   - GPIO：按键与 DS18B20 数据脚配置。

   - PinMux：D3 复用为 PWM；IO1 复用为 GPIO。

   - PWM：打开硬件 PWM，默认占空比 0%。

   - BLE：初始化 RN4020，建立自定义服务与可写特征。

  

2. 运行阶段（主循环）

   - 输入：BLE 命令（优先）→ 按键边沿/长按 → 自动模式温度采样。

   - 状态：

     - `led_enabled`：风扇使能开关（1 输出当前目标占空比；0 强制 0%）

     - `auto_mode`：模式标志（0 手动，1 自动）

     - `duty_idx`：手动档位索引（0/1/2 对应 30/80/100）

   - 输出：调用 PWM 驱动下发目标占空比。

  

控制策略关键点：

- “调速”完全交给硬件 PWM 外设，主循环只做事件识别与参数下发。

- BLE 与按键的逻辑统一映射到相同状态变量，避免出现两套控制逻辑互相打架。

  

## 四、各单元模块设计

  

### 4.1 初始化模块（`initial()`）

  

职责：完成各外设就绪与安全默认状态。

  

- GPIO

  - 打开按键与温度脚的位掩码。

  - 按键输入方向配置为输入。

  - DS18B20 总线默认释放（输入态）。

- 引脚复用

  - D3 切换为 PWM 功能。

  - IO1 切换为 GPIO 功能。

- PWM

  - 获取 PWM 设备并 `open`。

  - 默认写入 0% 占空比。

- BLE

  - 初始化 RN4020，并在失败时降级为“仅按键可用”。

  

### 4.2 PWM 输出模块（`pwm_apply_duty()`）

  

职责：将 0～100% 占空比写入硬件 PWM。

  

- 输入：`duty_percent`（0～100，超限钳位）。

- 输出：`pwm_timer_write(ch=0, mode=PWM, freq=1000Hz, duty=duty_percent)`。

- 设计理由：采用固定频率 1kHz，兼顾听感与控制分辨率；占空比调整不占用 CPU 周期。

  

### 4.3 按键事件识别模块（主循环内逻辑）

  

职责：在 1ms 轮询节拍下实现消抖、短按与长按，并映射为状态变更。

  

- 边沿检测：记录上一拍电平 `key*_prev` 与当前拍 `key*_now`。

- 计时方式：按住期间 `press_ms++`；释放时根据阈值判定有效短按。

- 参数：

  - 消抖阈值 `DEBOUNCE_MS=20ms`

  - 长按阈值 `LONG_PRESS_MS=1000ms`

- 特别处理：

  - 长按触发后置位 `key1_long_handled`，保证一次按住只触发一次开关动作。

  

### 4.4 DS18B20 采集与温控映射模块

  

#### 4.4.1 1-Wire 时序层（`onewire_*`）

  

职责：实现 DS18B20 1-Wire 物理层时序。

  

- 复位/Presence：`reset -> release -> sample`。

- 写 0/1：通过拉低时间差实现。

- 读位：短拉低后释放，在规定时间点采样总线。

- 延时：使用 `board_get_cur_us()` 实现微秒级忙等，保证时序满足 DS18B20 要求。

  

#### 4.4.2 温度读取（`ds18b20_read_temp_tenths()`）

  

流程：

1. 复位并确认 Presence。

2. `SKIP_ROM` + `CONVERT_T` 启动转换。

3. 等待约 750ms（12bit 最大转换时间）。

4. 再次复位，`SKIP_ROM` + `READ_SCRATCH` 读取前两个字节温度。

5. 计算：`temp_tenths = raw_temp * 10 / 16`。

  

异常处理：Presence 失败或读写异常返回错误码，主循环打印告警并跳过本次更新。

  

#### 4.4.3 温度到占空比映射（`duty_from_temp_tenths()`）

  

- 采用分段线性：低于 10℃ 输出 0%，高于 30℃ 输出 100%，中间区间线性插值。

- 优点：实现简单、行为可预期，适合作为示例与后续扩展基础。

  

### 4.5 BLE 远程控制模块（RN4020）

  

#### 4.5.1 初始化（`ble_init()`）

  

职责：设置设备名、启用自定义服务与可写特征，并进入可接收命令状态。

  

- 设备名：`SmartFan`。

- 服务：`RN4020_SERVICE_USER_DEFINED`。

- 特征：自定义 UUID，属性包含可写/写无响应/可读。

- 复位：通过 `rn4020_reset()` 使配置生效。

- 广播：调用 `rn4020_advertise()`，若返回错误仍允许系统继续运行（兼容部分固件状态差异）。

  

#### 4.5.2 命令接收与解析（`ble_poll_command()`）

  

职责：以非阻塞方式从 UART 收字节，兼容不同写入格式，并输出统一命令值。

  

支持三类输入：

1. RN4020 控制台上报行：`WV,xxxx,DATAHEX`（优先解析）

2. 透传文本行：`"1\n"/"2\n"/"3\n"`

3. 无换行单字节：`'1'/'2'/'3'`（仅在行首触发，降低误判）

  

连接管理：检测到 `Connection End` 时主动补发一次 `advertise` 以便手机快速重连。

  

### 4.6 统一控制状态机（`pwm_control()`）

  

职责：将 BLE/按键/温度三类输入统一汇聚，维护运行状态并驱动 PWM 输出。

  

- 优先级：BLE 命令 > 按键事件 > 自动模式温度采样。

- 行为要点：

  - `led_enabled=0` 时无论模式与档位如何都输出 0%。

  - 切换到自动模式后重置采样计数，尽快进行一次温控更新。

  - 手动模式下切档仅影响 `duty_idx`，是否立即生效取决于 `led_enabled`。

  

## 五、总结与体会

  

- 采用硬件 PWM 替代软件翻转 IO，可以显著降低 CPU 占用并提升输出稳定性，是执行器控制的工程化首选。

- 将“本地按键”和“远程 BLE”统一抽象为同一组状态变量与事件入口，能够避免多控制源导致的冲突，也便于后续扩展（例如增加串口命令、上位机等）。

- 裸机环境下的按键消抖与长按识别，用 1ms 轮询节拍实现直观可靠，但要注意长按只触发一次的“锁存处理”。

- DS18B20 的 1-Wire 时序对微秒级延时要求明确，使用板级计时接口实现忙等能保证时序正确；若未来系统引入更多任务，可进一步考虑用定时器/中断减少忙等时间。

- BLE 侧兼容多种写入格式（控制台行/透传行/单字节）能显著提高联调成功率，特别是在不同手机/小程序 BLE SDK 行为存在差异时。
