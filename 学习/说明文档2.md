# SmartFan项目设计说明

  

## 一、设计背景

  

本项目是一个基于ARC处理器的智能风扇控制系统，旨在实现一个集成本地控制、温度传感和无线通信的综合性嵌入式应用。随着物联网技术的快速发展，智能家居设备对智能化、节能化和远程控制的需求日益增长。传统的风扇控制系统往往功能单一，缺乏与环境互动和远程控制的能力。

  

本项目基于embARC OSP（Open Software Platform）嵌入式开发框架，在IoTDK（IoT Development Kit）开发板上实现了一个完整的智能风扇控制系统。系统通过硬件PWM精确控制风扇转速，外接DS18B20温度传感器实现自动温控，并利用RN4020蓝牙模块支持手机远程控制。该系统不仅可作为智能家居设备的原型，也可作为嵌入式系统学习的综合案例，涵盖了GPIO、PWM、UART、单总线通信等多种外设接口的应用。

  

## 二、设计要求

  

1. **硬件PWM调速**：使用硬件PWM外设实现风扇转速的精确控制，支持30%、80%、100%三档调速，避免使用软件模拟PWM带来的CPU占用和精度问题。

  

2. **双模式控制**：

   - 手动模式：用户可通过本地按键控制风扇档位和开关

   - 自动模式：根据环境温度自动调节风扇转速，实现智能化节能控制

  

3. **多输入源控制**：

   - 本地按键控制：两个物理按键分别实现档位切换和模式切换功能

   - 蓝牙远程控制：通过RN4020蓝牙模块支持手机小程序发送控制命令

  

4. **温度传感与自动控制**：

   - 集成DS18B20数字温度传感器，精度0.1℃

   - 实现温度-转速线性映射算法：温度≤10℃时风扇停止，温度≥30℃时全速运转，10-30℃之间线性调节

  

5. **系统实时性**：

   - 主循环以1ms固定节拍运行，确保按键响应和温度采样的实时性

   - 支持按键消抖（20ms）和长按识别（1000ms）

  

6. **可靠性与稳定性**：

   - 完善的错误处理机制，对传感器故障、通信异常等情况进行适当处理

   - 上电默认关闭风扇输出，确保安全

  

## 三、总体方案设计

  

### 3.1 系统架构

  

系统采用分层架构设计，分为硬件抽象层、驱动层和应用层：

  

1. **硬件抽象层**：通过embARC框架提供统一的硬件访问接口，包括GPIO、PWM、UART等外设

2. **驱动层**：实现DS18B20单总线驱动和RN4020蓝牙通信驱动

3. **应用层**：智能风扇控制逻辑，包括状态管理、事件处理和算法实现

  

### 3.2 硬件平台

  

- **开发板**：IoTDK (IoT Development Kit) 版本10

- **处理器核心**：ARC EM9D (arcem9d)

- **外设接口**：

  - PWM输出：Arduino D3引脚（PWM0通道）

  - 按键1输入：GPIO IO2（高电平有效）

  - 按键2输入：GPIO IO0（高电平有效）

  - 温度传感器：DS18B20使用Arduino IO1引脚（单总线）

  - 蓝牙模块：RN4020使用UART3接口

  

### 3.3 软件架构

  

系统采用事件驱动的状态机设计，主循环以1ms为节拍，按优先级处理各类事件：

  

1. **蓝牙命令处理**（最高优先级）：实时响应远程控制命令

2. **本地按键处理**：检测按键边沿事件，实现短按、长按功能

3. **自动模式控制**：定时采样温度并更新PWM占空比

  

### 3.4 控制逻辑

  

系统维护以下状态变量：

- `duty_idx`：手动模式下的档位索引（0-2对应30%、80%、100%）

- `auto_mode`：模式标志（0=手动，1=自动）

- `led_enabled`：风扇使能状态（1=开启，0=关闭）

- 按键状态机：记录按键按下时间，实现消抖和长按识别

  

## 四、各单元模块设计

  

### 4.1 硬件初始化模块（initial函数）

  

**功能**：完成所有硬件外设的初始化配置

  

**实现要点**：

1. GPIO初始化：配置按键和温度传感器引脚为输入模式

2. 引脚复用配置：将Arduino D3切换到PWM功能，IO1切换到GPIO功能

3. PWM设备初始化：打开PWM设备，设置默认占空比为0%

4. 蓝牙模块初始化：配置RN4020设备名称、服务和特征UUID

  

**关键代码**：

```c

// 引脚复用配置

result = io_arduino_config(FAN_PWM_ARDUINO_PIN, ARDUINO_PWM, IO_PINMUX_ENABLE);

result = io_arduino_config(FAN_TEMP_ARDUINO_PIN, ARDUINO_GPIO, IO_PINMUX_ENABLE);

  

// PWM设备初始化

fan_pwm = pwm_timer_get_dev(FAN_PWM_DEV_ID);

fan_pwm->pwm_timer_open();

pwm_apply_duty(0U);  // 默认关闭风扇

```

  

### 4.2 PWM控制模块（pwm_apply_duty函数）

  

**功能**：设置硬件PWM的占空比

  

**实现要点**：

1. 输入参数钳位：确保占空比在0-100%范围内

2. 调用PWM设备写接口：设置PWM模式、频率（1kHz）和占空比

3. 错误处理：对PWM写入失败进行警告输出

  

**关键代码**：

```c

if (duty_percent > 100U) {

    duty_percent = 100U;

}

ret = fan_pwm->pwm_timer_write(FAN_PWM_CH, DEV_PWM_TIMER_MODE_PWM,

                               FAN_PWM_FREQ_HZ, duty_percent);

```

  

### 4.3 温度传感模块（DS18B20驱动）

  

**功能**：读取DS18B20温度传感器的测量值

  

**实现要点**：

1. **单总线时序控制**：精确实现复位、写位、读位等基本操作

2. **温度转换流程**：

   - 发送跳过ROM命令（0xCC）

   - 发送温度转换命令（0x44），等待750ms转换完成

   - 再次复位总线，发送读取暂存器命令（0xBE）

   - 读取温度数据（2字节）

3. **温度值转换**：将原始数据转换为0.1℃单位的温度值

  

**关键代码**：

```c

// 温度读取流程

onewire_reset_presence();

onewire_write_byte(DS18B20_CMD_SKIP_ROM);

onewire_write_byte(DS18B20_CMD_CONVERT_T);

board_delay_ms(750U, 1);  // 等待转换完成

  

onewire_reset_presence();

onewire_write_byte(DS18B20_CMD_SKIP_ROM);

onewire_write_byte(DS18B20_CMD_READ_SCRATCH);

lsb = onewire_read_byte();

msb = onewire_read_byte();

  

// 温度值计算（0.1℃单位）

raw_temp = (int16_t)(((uint16_t)msb << 8) | lsb);

*temp_tenths = ((int32_t)raw_temp * 10) / 16;

```

  

### 4.4 温度-占空比映射模块（duty_from_temp_tenths函数）

  

**功能**：根据温度值计算PWM占空比

  

**算法设计**：

- 温度≤10℃：占空比=0%（风扇停止）

- 温度≥30℃：占空比=100%（全速运转）

- 10℃<温度<30℃：线性插值计算占空比

  

**数学公式**：

```

duty = (temp - 100) × 100 / (300 - 100)   [temp单位：0.1℃]

```

  

**关键代码**：

```c

if (temp_tenths <= TEMP_LOW_C_TENTHS) {      // ≤10℃

    return 0U;

}

if (temp_tenths >= TEMP_HIGH_C_TENTHS) {     // ≥30℃

    return 100U;

}

// 10-30℃之间线性计算

return (uint32_t)(((temp_tenths - TEMP_LOW_C_TENTHS) * 100) /

                  (TEMP_HIGH_C_TENTHS - TEMP_LOW_C_TENTHS));

```

  

### 4.5 蓝牙通信模块（RN4020驱动）

  

**功能**：实现蓝牙命令的接收和解析

  

**实现要点**：

1. **初始化配置**：设置设备名称、自定义服务UUID、特征UUID

2. **非阻塞数据接收**：使用ez_sio库实现串口数据的非阻塞读取

3. **多格式命令解析**：

   - RN4020格式：解析"WV,xxxx,HEX"格式的数据包

   - 透传文本格式：解析"1"、"2"、"3"等简单命令

   - 单字节格式：兼容无换行符的单字节命令

4. **命令映射**：

   - 命令"1"：开关风扇（toggle）

   - 命令"2"：切换档位（30%→80%→100%→30%）

   - 命令"3"：切换手动/自动模式

  

**关键代码**：

```c

// 命令解析逻辑

if (strncmp(line, "WV,", 3) == 0) {

    // 解析RN4020格式：WV,xxxx,HEX

    data_ptr = line + 8;

    ble_hex_to_u8(data_ptr[0], data_ptr[1], &cmd);

} else if (line_len >= 1U) {

    // 解析透传文本格式

    if (line_buf[0] == '1' || line_buf[0] == '2' || line_buf[0] == '3') {

        cmd = (uint8_t)line_buf[0];

    }

}

```

  

### 4.6 主控制循环模块（pwm_control函数）

  

**功能**：系统主状态机，协调各模块工作

  

**实现要点**：

1. **事件处理优先级**：

   - 蓝牙命令（最高优先级，保证远程控制实时性）

   - 本地按键事件

   - 自动模式温度采样

2. **按键状态机**：

   - 按键消抖：按下时间≥20ms才视为有效短按

   - 长按识别：按下时间≥1000ms触发长按事件

   - 边沿检测：通过比较当前拍和上一拍的电平变化

3. **模式管理**：

   - 手动模式：按键1短切换档位，长按开关风扇

   - 自动模式：定时采样温度，自动计算并更新占空比

4. **状态同步**：确保本地控制和蓝牙控制的状态一致性

  

**关键设计**：

```c

// 主循环结构

while (1) {

    // 1. 处理蓝牙命令（最高优先级）

    ble_cmd = ble_poll_command();

    if (ble_cmd == '1') { ... }

    else if (ble_cmd == '2') { ... }

    else if (ble_cmd == '3') { ... }

    // 2. 读取按键状态（高电平表示按下）

    key_gpio->gpio_read(&key_val, KEY_IO2_MASK | KEY2_IO0_MASK);

    key1_now = (key_val & KEY_IO2_MASK) ? 1 : 0;

    key2_now = (key_val & KEY2_IO0_MASK) ? 1 : 0;

    // 3. 处理按键2（模式切换）

    if ((key2_prev == 0) && (key2_now == 1)) { ... }  // 上升沿

    if ((key2_prev == 1) && (key2_now == 0)) { ... }  // 下降沿

    // 4. 处理按键1（档位切换/开关）

    if ((key1_prev == 0) && (key1_now == 1)) { ... }  // 上升沿

    if (key1_now == 1) { ... }  // 长按处理

    if ((key1_prev == 1) && (key1_now == 0)) { ... }  // 下降沿

    // 5. 自动模式温度控制

    if (auto_mode) {

        auto_sample_ms++;

        if (auto_sample_ms >= AUTO_SAMPLE_MS) {

            ds18b20_read_temp_tenths(&temp_tenths);

            duty = duty_from_temp_tenths(temp_tenths);

            pwm_apply_duty(duty);

        }

    }

    // 6. 更新状态，等待下一拍

    key1_prev = key1_now;

    key2_prev = key2_now;

    board_delay_ms(1, 1);  // 1ms节拍

}

```

  

## 五、总结与体会

  

### 5.1 项目总结

  

本项目成功实现了一个功能完整的智能风扇控制系统，具备以下特点：

  

1. **硬件资源充分利用**：通过硬件PWM实现精确的风扇调速，避免了软件模拟PWM的CPU占用问题；充分利用GPIO、UART、单总线等多种接口。

  

2. **多控制方式集成**：将本地按键控制、温度自动控制和蓝牙远程控制有机整合，提供了灵活多样的操作方式。

  

3. **实时性保障**：1ms的主循环节拍确保了系统响应的实时性，按键消抖和长按识别功能提升了用户体验。

  

4. **鲁棒性设计**：完善的错误处理机制和状态管理逻辑，提高了系统的稳定性和可靠性。

  

5. **可扩展性**：模块化的代码结构便于功能扩展和维护，可作为更复杂智能家居系统的基础。

  

### 5.2 遇到的问题与解决方案

  

#### 问题1：PWM控制方式的改进

  

**问题描述**：项目初期尝试使用Arduino数字接口通过软件模拟PWM输出，但发现效果不理想。

  

**问题分析**：

- 软件模拟PWM需要CPU频繁翻转GPIO电平，占用大量CPU时间

- 模拟PWM的频率和占空比精度受限于软件延时精度

- 在高频PWM需求下，软件模拟可能导致系统响应延迟

  

**解决方案**：

- 改用硬件PWM外设，利用ARC处理器的PWM定时器模块

- 硬件PWM优点：

  - 不占用CPU时间，由硬件自动生成PWM波形

  - 频率和占空比精度高，稳定性好

  - 支持更高的PWM频率（本项目采用1kHz）

- 代码修改：

  ```c

  // 从软件模拟改为硬件PWM

  result = fan_pwm->pwm_timer_write(FAN_PWM_CH, DEV_PWM_TIMER_MODE_PWM,

                                   FAN_PWM_FREQ_HZ, duty_percent);

  ```

  

#### 问题2：按键电平逻辑的调整

  

**问题描述**：初期设计中使用低电平（0）表示按键按下，高电平（1）表示按键释放。但在实际测试中发现，按键按下时GPIO读取到的电平依然为高电平。

  

**问题分析**：

- 开发板的按键电路设计为按下时接通上拉电阻，导致按键按下时GPIO读取到高电平

- 初始设计假设与硬件实际电路不符

- 需要根据实际硬件电路调整软件逻辑

  

**解决方案**：

- 修改按键电平逻辑：高电平（1）表示按键按下，低电平（0）表示按键释放

- 调整按键状态检测代码：

  ```c

  // 修改前：低电平表示按下

  key1_now = (key_val & KEY_IO2_MASK) ? 0 : 1;

  // 修改后：高电平表示按下

  key1_now = (key_val & KEY_IO2_MASK) ? 1 : 0;

  ```

- 同步修改所有按键处理逻辑，确保边沿检测和状态判断正确

  

### 5.3 经验与体会

  

通过本项目的开发，获得了以下宝贵的嵌入式系统开发经验：

  

1. **硬件-软件协同设计的重要性**：在嵌入式系统开发中，必须充分考虑硬件特性。按键电平逻辑的调整经验表明，软件设计必须基于实际的硬件电路，不能仅凭假设或常见设计模式。

  

2. **外设选择的考量**：硬件PWM与软件模拟PWM的选择体现了性能与资源占用的权衡。对于实时性要求高的控制任务，应优先使用硬件外设以释放CPU资源。

  

3. **系统实时性设计**：1ms的主循环节拍设计确保了系统的响应速度，这种固定节拍的设计模式在嵌入式实时系统中具有很好的参考价值。

  

4. **错误处理与鲁棒性**：完善的错误处理机制（如传感器读取失败处理、通信异常处理）是工业级嵌入式系统必备的特性。

  

5. **模块化与可维护性**：清晰的模块划分和函数封装大大提高了代码的可读性和可维护性，便于后续功能扩展和调试。

  

6. **多控制源集成**：成功将本地控制、传感器反馈和无线通信集成到一个统一的状态机中，这种设计模式在物联网设备开发中具有普遍适用性。

  

本项目不仅实现了一个功能完整的智能风扇控制系统，更通过实践深化了对嵌入式系统设计原则的理解，为后续更复杂的嵌入式项目开发奠定了坚实基础。
